# Signal Schedule Documentation

Technical planning, foundation, migration, audit, engine, and UI notes live here so the repository root stays focused on deployable app files and core project handoff documents.

## Folders

- `audits/` — QA, drift, and integration audit notes.
- `architecture/` — repository and file organization notes.
- `engines/` — scheduling, staffing, request, and history engine foundations.
- `foundation/` — feature foundation documents.
- `hotfixes/` — focused hotfix notes retained for traceability.
- `migrations/` — migration notes, including standalone repository migration.
- `releases/` — current release summary.
- `ui/` — UI-specific planning notes.
